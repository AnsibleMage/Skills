# CREATE MODE v1.0 — 제로 입력 → 완성 랜딩페이지

> 와이어프레임 없이 주제/업종만으로 랜딩페이지를 처음부터 생성.
> An & Ari의 카피 파이프라인 + 슬롯 엔진 시각 다양성 합체.

---

## 트리거 조건

```
다음 중 하나라도 해당하면 CREATE 모드:
- 와이어프레임/HTML 파일이 입력되지 않음
- "만들어줘", "생성해줘", "새로 만들어줘" 동사
- 업종/주제만 제시 ("인테리어 랜딩페이지", "카페 웹사이트")
- "처음부터", "from scratch" 키워드
```

## STEP 0: 콘텐츠 생성 파이프라인

### 0a. 도메인 분석 (Domain Analysis)

```yaml
입력: 사용자 프롬프트 (예: "인테리어 웹사이트 랜딩페이지 만들어줘")

추출 항목:
  industry: "인테리어/건축"
  target_audience: "신혼부부, 리모델링 고객, 30~40대"
  tone: "고급, 신뢰, 감성"
  key_services: ["주거 인테리어", "상업 공간", "리모델링", "컨설팅"]
  differentiator: "디자인 감각 + 시공 품질"
```

**도메인별 자동 매핑 테이블:**

| 업종 | 핵심 섹션 | 톤 | 추천 SLOT_1 | 추천 SLOT_2 |
|------|---------|-----|-----------|-----------|
| 인테리어/건축 | 포트폴리오, 프로세스, 시공사례 | 고급/감성 | 갤러리화이트큐브, 일본정원 | 웜크림, 오크 |
| IT/SaaS | 기능, 가격, 데모 | 테크/신뢰 | 글래스스튜디오, 다크룸 | 쿨그레이, 딥네이비 |
| 교육/코칭 | 커리큘럼, 강사, 후기 | 친근/전문 | 북바인더리, 무인양품 | 웜크림, 라벤더 |
| 뷰티/헬스 | Before/After, 성분, 후기 | 감성/신뢰 | 온실, 미술관카페 | 로즈, 블러쉬 |
| F&B/카페 | 메뉴, 공간, 스토리 | 따뜻/감성 | 미술관카페, 레코드숍 | 웜크림, 앰버 |
| 부동산 | 매물, 지도, 상담 | 신뢰/정보 | 건축사무소, 지도제작실 | 순백, 슬레이트블루 |
| 법률/회계 | 서비스, 전문가, 상담 | 권위/신뢰 | 약국서랍장, 활판인쇄소 | 순백, 스틸 |
| 패션/쇼핑몰 | 상품, 룩북, 카테고리 | 트렌디/대담 | 네온사인, 빈티지포스터 | 차콜, 미드나잇 |
| 비영리/공공 | 미션, 활동, 참여 | 따뜻/투명 | 온실, 무인양품 | 민트, 올리브 |
| 제조/산업 | 제품, 스펙, 공정 | 견고/기술 | 공사현장, 실험실 | 스틸, 콘크리트 |

### 0b. 섹션 구조 자동 생성 (Section Blueprint)

> 업종별 최적 섹션 조합을 자동 결정.

**홈페이지/랜딩페이지 섹션 풀 (25개 — 헤더+푸터 필수, 본문 6~10개 선택):**

#### 필수 섹션 (항상 포함)

| # | 섹션 | 역할 | HTML 요소 |
|---|------|------|----------|
| **H** | **Header/Navigation** | 로고 + 메뉴 + 사용자 영역 | `<header>` / `<nav>` |
| **F** | **Footer** | 회사 정보 + 링크 + 저작권 + SNS | `<footer>` |

#### 본문 섹션 풀 (25개 중 6~10개 선택)

| # | 섹션 | 역할 | 포함 요소 | 적합 업종 |
|---|------|------|---------|---------|
| S01 | **Hero Banner** | 첫인상. 핵심 메시지 + CTA | 대형 제목 + 부제 + CTA 버튼 + 배경(이미지/그라디언트) | 전체 |
| S02 | **검색/AI 큐레이터** | 핵심 기능 즉시 접근 | 검색바 + 추천 태그 + AI 안내 문구 | 교육, 포털, 커머스 |
| S03 | **서비스 소개** | 핵심 서비스 3~6개 | 아이콘/썸네일 + 제목 + 1줄 설명 카드 그리드 | 전체 |
| S04 | **기능/특징** | 제품 핵심 기능 | 기능 카드 + 스크린샷/일러스트 | SaaS, 앱, 테크 |
| S05 | **포트폴리오/갤러리** | 작업물 시각 전시 | 이미지 그리드 + 카테고리 필터 + 제목+설명 | 인테리어, 디자인, 사진 |
| S06 | **콘텐츠 카드 행** | 콘텐츠 브라우징 | 가로 스크롤 카드 행 + 썸네일 + 뱃지 + 메타 | 교육, 미디어, 커머스 |
| S07 | **프로세스/진행과정** | 이용 절차 안내 | 3~5단계 스텝 + 번호 + 아이콘 + 설명 | 서비스, 컨설팅, 제조 |
| S08 | **실적/숫자** | 신뢰 수치 | 큰 숫자 3~5개 + 라벨 + 단위 | 전체 |
| S09 | **고객 후기** | 사회적 증거 | 인용문 + 이름 + 직함 + 별점/결과 | 전체 |
| S10 | **팀/회사 소개** | 신뢰 구축 | 팀원 카드 또는 회사 스토리 + 미션 | 전체 |
| S11 | **파트너/클라이언트** | 권위 증명 | 로고 배열 (마키 또는 그리드) | B2B, 제조, 기업 |
| S12 | **가격표** | 구매 결정 유도 | 2~3 플랜 카드 + 기능 비교 + CTA | SaaS, 교육, 구독 |
| S13 | **FAQ** | 구매 장벽 제거 | 아코디언 5~8개 | 전체 |
| S14 | **블로그/뉴스** | 최신 소식 | 포스트 카드 3~4개 + 제목+날짜+썸네일 | 미디어, 교육, 기업 |
| S15 | **이벤트/공지** | 긴급/시의성 | 배너 + 날짜 + 카운트다운 | 교육, 커머스, 공공 |
| S16 | **현황/대시보드** | 사용자 데이터 | 진행 차트 + pill 뱃지 + 포인트 | 교육, SaaS, 포털 |
| S17 | **구독/채널** | 커뮤니티 연결 | 채널 카드 + 아바타 + 구독자 수 | 교육, 미디어 |
| S18 | **상품/메뉴** | 판매 아이템 | 상품 카드 + 가격 + 썸네일 + 장바구니 | F&B, 커머스 |
| S19 | **도서/리소스** | 추천 콘텐츠 | 표지 + 제목 + 저자 가로 스크롤 | 교육, 도서관, 미디어 |
| S20 | **Before/After** | 변화 증명 | 좌우 비교 + 슬라이더 | 뷰티, 인테리어, 교육 |
| S21 | **지도/위치** | 오프라인 안내 | 지도 임베드 + 주소 + 교통 | 매장, 기관, 학교 |
| S22 | **문의/연락처** | 소통 채널 | 전화 + 이메일 + 운영시간 카드 | 전체 |
| S23 | **문의 폼** | 직접 문의 | 이름 + 이메일 + 내용 입력폼 + 전송 버튼 | 서비스, 컨설팅 |
| S24 | **뉴스레터 구독** | 리드 수집 | 이메일 입력 + 구독 버튼 + 혜택 안내 | 미디어, 교육, 커머스 |
| S25 | **최종 CTA** | 마지막 행동 유도 | 강조 배경 + 대형 제목 + CTA 버튼 | 전체 |

#### Header(H) 구성 규칙

```yaml
header_structure:
  required:
    - logo: 브랜드명 텍스트 또는 로고 이미지
    - gnb: 글로벌 네비게이션 (4~6개 메뉴)
    - cta_or_user: CTA 버튼("시작하기") 또는 사용자 영역("로그인/회원가입")
  optional:
    - util_bar: 상단 유틸리티 바 (언어, 다크모드, 관리자 링크)
    - search: 헤더 내 검색 아이콘/바
    - notification: 알림 아이콘
  style:
    - SLOT_3 구조에 따라 위치 결정 (상단 바 / 사이드바 / 투명 오버레이)
    - 기본: position sticky, 스크롤 시 배경 blur 또는 색상 변화
    - 모바일: 햄버거 메뉴로 축소
```

#### Footer(F) 구성 규칙

```yaml
footer_structure:
  required:
    - brand: 로고/브랜드명 + 1줄 소개
    - links: 주요 링크 그룹 (서비스/회사/법적/고객지원)
    - legal: 저작권 + 개인정보처리방침 + 이용약관
    - contact: 전화/이메일/주소 (최소 1개)
  optional:
    - sns: SNS 아이콘 링크 (인스타/페북/유튜브)
    - newsletter: 이메일 구독 입력
    - sitemap: 전체 사이트맵 링크
  style:
    - 2~3 컬럼 또는 미니멀 1~2줄
    - "4컬럼 링크 팜" AI 클리셰 금지 → 차별화된 구조
    - 배경: 기본 bg보다 1~2단계 어둡게
```

### 0c. 카피라이팅 (Copywriting)

> 각 섹션별 텍스트를 자동 생성. An & Ari 카피 패턴 적용.

**카피 생성 규칙:**

```yaml
headline_formulas: # 5가지 중 1개 자동 선택
  - result_period: "[구체적 결과]를 [기간] 만에"
  - problem_solve: "[문제] 없이 [원하는 결과]"
  - target_benefit: "[타겟]을 위한 [핵심 혜택]"
  - number_proof: "[숫자]명이 선택한 [서비스]"
  - comparison: "[기존 방법] 대신 [새로운 방법]으로"

copy_rules:
  - 한국어 자연스러운 구어체 (번역투 금지)
  - 감정 → 논리 흐름
  - 구체적 숫자 ("많은" 대신 "1,247명")
  - 짧은 문장 (20자 내외)
  - 라운드 넘버 금지 (50,000 → 47,200)

per_section:
  header:
    logo: 브랜드명 (사용자 입력 또는 도메인에서 자동 생성)
    gnb: 4~6개 메뉴명 (서비스/소개/포트폴리오/고객후기/문의)
    cta: "시작하기" / "무료 상담" / "로그인"
  hero:
    headline: 3가지 옵션 중 최적 선택
    subheadline: 타겟 + 방법 힌트
    cta: 행동 유도 문구
  services:
    각 서비스별: 제목(8자 내) + 설명(30자 내)
  content_row:
    섹션 제목 + "전체보기" 링크 + 카드 3~5개 (썸네일+제목+메타)
  testimonials:
    3개 후기: 이름(가명) + 직업 + 구체적 결과 + 감정 표현
  stats:
    4개 수치: 구체적 숫자 + 단위 + 라벨
  contact:
    3~5개 연락 수단: 아이콘 + 라벨 + 값 (전화/이메일/주소/운영시간)
  footer:
    brand: 로고 + 1줄 소개
    link_groups: 2~3 그룹 (서비스/회사/법적)
    legal: "개인정보처리방침 | 이용약관 | 저작권 ⓒ 2026"
    contact_mini: 전화/이메일/주소 1줄
```

## CREATE 모드 전체 플로우

```
사용자: "인테리어 랜딩페이지 만들어줘"

[0b] 섹션 선택 → H(헤더) + [Hero, 포트폴리오, 프로세스, B/A, 서비스, 후기, 실적, 연락처, CTA] + F(푸터)
    ↓
[0c] 카피 생성 → 헤더 메뉴명 + 각 섹션 텍스트 + 푸터 정보
    ↓
[0d] 내부 와이어프레임 JSON 생성 (H~F 포함)
    ↓
[기존 STEP 1~7] SLOT 엔진 + 품질 모듈 적용 → HTML 생성
```
