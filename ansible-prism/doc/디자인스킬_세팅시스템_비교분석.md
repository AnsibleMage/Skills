# 디자인 스킬 세팅 시스템 비교 분석

> "세팅이 이런류의 스킬에 가장 중요한 로직이다" — 앤의 인사이트
>
> 두 스킬의 세팅(Configuration) 구조를 비교하여,
> AI가 만든 느낌을 없애고 사람이 만든 것처럼 보이게 하는 핵심 메커니즘을 분석한다.

---

## 1. 비교 대상

| 항목 | landing-page-generator | supanova-forge |
|------|----------------------|----------------|
| 출력 | Gemini API → 이미지(PNG) 13장 스티칭 | HTML + Tailwind CSS 단일 파일 |
| 세팅 방식 | 제품 속성 입력 → 프리셋 자동 매핑 | 사용자 직접 숫자/옵션 선택 |
| 에이전트 | 5단계 파이프라인 (intake→research→copy→design→prompt) | 단일 스킬 (Configuration Interview → 생성) |

---

## 2. landing-page-generator의 세팅 구조

### 2.1 파이프라인 전체 흐름

```
01-intake        → 제품 정보 수집 (이름, 타겟, 문제, 혜택, 가격, 긴급성)
02-research      → 타겟/페인포인트 리서치
03-copy          → 13섹션 카피라이팅
04-design-direction → ★ 스타일 프리셋 결정 (핵심 로직)
05-prompt-generator → Gemini 이미지 프롬프트 생성
```

### 2.2 design-direction 에이전트의 결정 로직 (04-design-direction.md)

```
입력: 제품 가격대 + 타겟 특성 + 긴급성 강도 + 브랜드 컬러

결정 로직:
  가격대 == "고가"    → premium
  긴급성 == "높음"    → sales
  타겟 == "전문가"    → minimal
  그 외              → community
```

### 2.3 4개 프리셋 — 완전한 디자인 시스템

| 프리셋 | 컬러 | radius | 그림자 | 폰트 톤 | 여백 | 아이콘 | 적합 |
|--------|------|--------|--------|---------|------|--------|------|
| **minimal** | Blue #2563EB | 8px | subtle | Bold 깔끔 | 여유 | line | SaaS, B2B |
| **sales** | Red #DC2626 + Yellow | 0-4px | 강한 | Black 아주크게 | 빡빡 | filled | 이벤트, 프로모션 |
| **premium** | Dark #1F2937 + Gold | 0-4px | 미묘 | Serif Medium | 넉넉 | outline | 고가, 럭셔리 |
| **community** | Purple #7C3AED + Pink | 12-16px | 부드러운 | SemiBold 둥근 | 보통 | rounded | 교육, 커뮤니티 |

### 2.4 프리셋이 제어하는 하위 속성 (예: premium)

```
premium 선택 시 자동 연동:
├── color_palette
│   ├── primary: #1F2937
│   ├── accent: #D4AF37 (Gold)
│   ├── background: #F9FAFB
│   └── text: #111827
├── typography
│   ├── headline: Medium/SemiBold (Bold 아님)
│   ├── font_style: Serif 고려
│   └── letter_spacing: 넉넉한 자간
├── components
│   ├── button.radius: 0-4px
│   ├── button.shadow: 미묘한 그림자
│   ├── card.radius: 12px
│   └── icon_style: outline
├── layout
│   ├── section_gap: 100px
│   ├── inner_padding: 48px
│   └── alignment: center
└── section_backgrounds (13개 섹션 전부 사전 정의)
    ├── hero: primary gradient
    ├── pain: background_alt
    ├── ...
    └── final_cta: primary gradient
```

**핵심: 프리셋 하나가 30+ 속성을 일관되게 변경한다.**

## 3. supanova-forge의 세팅 구조

### 3.1 Configuration Interview

```
1. 페이지 목적: conversion | brand | portfolio | saas | ecommerce
2. 레이아웃 실험성: 1~10 (1=대칭, 10=비대칭+여백)
3. 애니메이션 수준: 1~10 (1=호버만, 10=스크롤 트리거)
4. 콘텐츠 밀도: 1~10 (1=럭셔리 여백, 10=데이터 밀집)
5. 분위기(VIBE): dark | warm | clean
```

### 3.2 VIBE 3가지 아키타입

| VIBE | 배경 | 강조색 | 질감 | 분위기 |
|------|------|--------|------|--------|
| **Vantablack Luxe** (dark) | OLED black #050505 | mesh gradient | glass+blur | 테크/SaaS |
| **Warm Editorial** (warm) | cream #FDFBF7 | sage/espresso | noise overlay | 라이프/브랜드 |
| **Clean Structural** (clean) | silver-grey | display typography | ambient shadow | 헬스/포트폴리오 |

### 3.3 독립 슬라이더

```
DESIGN_VARIANCE (1-10): Hero 레이아웃, 섹션 비대칭도
MOTION_INTENSITY (1-10): 애니메이션 복잡도
VISUAL_DENSITY (1-10): 여백 대비 콘텐츠 양
```

## 4. 핵심 차이 분석

### 4.1 세팅 철학

| 관점 | landing-page-generator | supanova-forge |
|------|----------------------|----------------|
| 접근 | **문맥 기반 자동 판단** | **수동 세밀 제어** |
| 세팅 수 | 1개 (프리셋 선택) | 5개 (목적+VIBE+숫자3개) |
| 하위 속성 연동 | 30+ 속성 일괄 변경 | VIBE만 연동, 나머지 독립 |
| 일관성 보장 | 프리셋 내부에서 보장 | 조합에 따라 불일치 가능 |

### 4.2 왜 landing-page-generator가 "사람 느낌"인가

사람 디자이너의 작업 방식:
```
"이 프로젝트는 premium 톤이야"
    ↓
  컬러, 폰트, 여백, 보더, 그림자가 자연스럽게 따라옴
    ↓
  전체적으로 일관된 디자인 시스템 적용
```

landing-page-generator는 이 과정을 **프리셋 연동으로 모방**:
```
가격 고가 + 타겟 전문가 → premium 자동 선택
    ↓
  30+ 속성이 premium 세트로 일괄 적용
    ↓
  13개 섹션 전체에 동일한 디자인 시스템 관통
```

AI가 만든 느낌이 나는 이유:
```
VIBE: clean + DENSITY: 7 + VARIANCE: 4
    ↓
  "깨끗한데 밀도 높고 대칭적" → 속성 간 충돌
    ↓
  각 속성이 독립적으로 적용되어 전체가 어색
```

### 4.3 구체적 비교 (index_05 vs index_06)

| 속성 | index_05 (clean+밀도6) | index_06 (warm+밀도3) | 결과 |
|------|----------------------|---------------------|------|
| 여백 | py-10~12 | py-16~24 | 06이 숨 쉬는 느낌 |
| 배경 교차 | white→alt 단조 | ivory→dark→ivory | 06이 리듬감 |
| 타이포 범위 | 11px~24px (좁음) | 10px~7xl (극적) | 06이 시각 계층 명확 |
| 이미지 처리 | 컬러 그대로 | grayscale→컬러 호버 | 06이 세련됨 |
| 보더 | 1px flat | hover 시 warm 전환 | 06이 인터랙티브 |
| 노이즈 텍스처 | 없음 | CSS grain overlay | 06이 수작업 느낌 |

## 5. 개선 제안: supanova-forge에 프리셋 연동 도입

### 5.1 현재 (독립 슬라이더)

```
사용자 → VIBE: warm
사용자 → DENSITY: 7     ← warm인데 밀도 높으면 어색
사용자 → VARIANCE: 3    ← warm인데 대칭이면 editorial 느낌 안 남
```

### 5.2 개선안 (프리셋 연동 + 오버라이드)

```
사용자 → VIBE: warm
    ↓ 자동 연동
    DENSITY: 3 (넉넉한 여백)
    VARIANCE: 8 (비대칭, 드라마틱)
    RADIUS: 0 (날카로운 모서리)
    SHADOW: none (플랫, 보더만)
    BORDER: warm-tinted strong
    TEXTURE: noise overlay
    SECTION_RHYTHM: ivory→dark→ivory 교차
    ↓
사용자 → DENSITY: 5 (오버라이드 원할 시)
```

### 5.3 VIBE별 자동 연동 테이블

| 속성 | dark (기본) | warm | clean |
|------|------------|------|-------|
| DENSITY | 5 | 3 | 5 |
| VARIANCE | 8 | 8 | 6 |
| RADIUS | 8px | 0 | 4px |
| SHADOW | tinted glass | none (border only) | ambient diffused |
| BORDER | subtle white/10 | strong warm-tinted | 1px subtle |
| TEXTURE | mesh gradient | noise overlay | none |
| SECTION_BG | dark mono | light↔dark 교차 | white↔gray 교차 |
| FONT_DISPLAY | Grotesk wide | Serif/Display dramatic | Bold large |
| LETTER_SPACING | tight -0.02em | dramatic -0.04em | normal |
| HOVER_STYLE | glow/scale | grayscale→color | lift/shadow |

## 6. 결론

**"세팅 시스템이 디자인 품질의 핵심"이라는 앤의 판단은 정확하다.**

- landing-page-generator: **프리셋 1개 → 30+ 속성 일관 연동** → 사람이 만든 느낌
- supanova-forge: **VIBE + 독립 슬라이더** → 조합에 따라 AI 느낌 가능

개선 방향: VIBE 선택 시 하위 속성이 자동 연동되고, 사용자는 필요시 개별 오버라이드만 하는 구조.
