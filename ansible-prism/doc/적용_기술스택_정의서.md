# 적용 기술스택 정의서 — Wireframe Skill

## 개요

Claude Code의 `/wireframe` 스킬은 UX 프로토타입을 2단계(Phase)로 생성한다. Phase 1에서 5개의 B&W 와이어프레임 옵션을 생성하고, Phase 2에서 5개 병렬 에이전트가 각 옵션의 Clean + Polished 색상 변형을 추가한다. **외부 의존성 없이** 순수 HTML + CSS + Vanilla JS로 동작하는 자립형(self-contained) 프로토타입을 출력한다.

---

## 1. 기술스택 구성

### 1.1 스택 요약

| 계층 | 기술 | 버전 | 역할 | 의존성 |
|------|------|------|------|--------|
| **마크업** | Vanilla HTML5 | — | 5개 옵션 탭 + 서브탭 + 브라우저 프레임 | 없음 (CDN 없음) |
| **스타일** | Pure CSS (3-layer) | CSS3 | base.css + styles.css + styles-optN.css | 없음 |
| **인터랙션** | Vanilla JavaScript | ES5 | 탭 전환, 서브탭 전환, 완료 배너 | 없음 (inline) |
| **아이콘** | Inline SVG | SVG 1.1 | Iconpark Thunder (sketch/paint/diamond) | 없음 (임베드) |
| **폰트** | System Font Stack | — | `-apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif` | 없음 |
| **폰트 (Phase 2)** | Google Fonts | — | Polished 변형에서 `@import`로 로드 | CDN (선택적) |
| **디자인 시스템** | design-taste.md + design-context.md | — | 스타일 토큰, 프로젝트별 컨텍스트 | Markdown 파일 |
| **에이전트** | Claude Code Task Agent | — | Phase 2: 5개 병렬 Visual Designer | Claude Code 런타임 |

### 1.2 Zero-Dependency 원칙

```
외부 CDN:          ✕ (Phase 1 완전 차단, Phase 2 Google Fonts만 허용)
프레임워크:         ✕ (React, Vue, Tailwind 등 없음)
아이콘 라이브러리:   ✕ (Inline SVG로 임베드)
빌드 도구:          ✕ (번들러, 트랜스파일러 없음)
```

출력 파일을 아무 브라우저에서 더블클릭만으로 즉시 열 수 있는 완전 자립형 구조.

## 2. CSS 아키텍처

### 2.1 3-Layer CSS 구조

```
<head>
  <link rel="stylesheet" href="base.css">         ← Layer 1: 프레임워크 (불변)
  <link rel="stylesheet" href="styles.css">        ← Layer 2: 와이어프레임 콘텐츠 (Phase 1)
  <link rel="stylesheet" href="styles-opt1.css">   ← Layer 3: 색상 변형 (Phase 2, 옵션별)
  <link rel="stylesheet" href="styles-opt2.css">
  <link rel="stylesheet" href="styles-opt3.css">
  <link rel="stylesheet" href="styles-opt4.css">
  <link rel="stylesheet" href="styles-opt5.css">
</head>
```

| Layer | 파일 | 작성 시점 | 라인 예산 | 역할 |
|-------|------|----------|----------|------|
| **1. Framework** | `base.css` | 템플릿 복사 (불변) | ~245줄 | 레이아웃, 탭, 브라우저 프레임, 어노테이션, 배너, 푸터 |
| **2. Content** | `styles.css` | Phase 1 (UX Architect) | ≤ 300줄 | 와이어프레임 콘텐츠 CSS (카드, 폼, 그리드 등) |
| **3. Variant** | `styles-optN.css` | Phase 2 (Visual Designer) | ≤ 200줄/파일 | Clean + Polished 색상 오버라이드 + self-reveal |

### 2.2 CSS Cascade 규칙

- Layer 3은 Layer 1/2를 **오버라이드만** 함 (레이아웃 중복 금지)
- 변형 CSS에서 허용되는 속성: `color`, `background`, `border-color`, `box-shadow`, `font-family`, `font-weight`, `transition`, `animation`
- 레이아웃 속성(`display`, `flex`, `grid`, `width`, `height`, `margin`, `padding`) 재선언 금지

### 2.3 Self-Reveal 메커니즘

Phase 2 에이전트가 `styles-optN.css`를 작성하면, 파일 끝에 self-reveal CSS가 삽입되어:

```css
/* Self-reveal: Phase 2 완료 시 자동 활성화 */
#frame-optN-tabs .sub-tab-badge { display: none; }
#frame-optN-tabs .sub-tab-btn[data-variant="clean"] { color: #64B5F6; }
#frame-optN-tabs .sub-tab-btn[data-variant="clean"]:hover { color: #1E88E5; }
#frame-optN-tabs .sub-tab-btn[data-variant="clean"].active { color: #1565C0; border-bottom-color: #1565C0; }
#frame-optN-tabs .sub-tab-btn[data-variant="polished"] { color: #81C784; }
#frame-optN-tabs .sub-tab-btn[data-variant="polished"]:hover { color: #43A047; }
#frame-optN-tabs .sub-tab-btn[data-variant="polished"].active { color: #2E7D32; border-bottom-color: #2E7D32; }
```

**원리**: 빈 CSS 파일 → "generating..." 배지 표시 + 서브탭 회색. CSS 작성 완료 → 배지 숨김 + 서브탭 색상 활성화. 페이지 재오픈 시 JS가 배지 표시 여부 체크 → 전체 완료 시 배너 표시.

### 2.4 CSS 클래스 전환 (Sub-tab)

```
.browser-frame.wireframe  → B&W 와이어프레임 (기본)
.browser-frame.clean      → Clean 색상 변형
.browser-frame.polished   → Polished 색상 변형
```

HTML은 동일 — wrapper 클래스만 JS로 토글. 어노테이션(`.wf-annotations`)은 `.clean`, `.polished`에서 `display: none` 처리.

## 3. 색상 시스템

### 3.1 Wireframe 팔레트 (Phase 1 — 엄격)

| 용도 | HEX | 허용 범위 |
|------|-----|----------|
| 텍스트/테두리 | `#000` | 주요 텍스트, 실선 테두리 |
| 보조 텍스트 | `#333` | 부제목, 설명 |
| 3차 텍스트 | `#666` | 라벨, 캡션 |
| 플레이스홀더 | `#999` | 비활성, 힌트 텍스트 |
| 테두리/구분선 | `#ccc` | 카드 테두리, 수평선 |
| 미세 배경 | `#eee` | hover, 서브 영역 |
| 배경 | `#fff` | 기본 배경 |

> Phase 1에서 위 7색 외 **모든 색상 금지**. 브랜드 컬러, 시맨틱 컬러 없음.

### 3.2 Clean 변형 (Phase 2)

- `design-context.md`의 프로젝트 팔레트 사용
- 시스템 폰트 유지 (`-apple-system` 스택)
- 솔리드 컬러만 (그래디언트 없음, 그림자 없음, 효과 없음)
- `design-taste.md`의 Warmth(소비자) 또는 Precision(관리자) 토큰 적용

### 3.3 Polished 변형 (Phase 2)

- Clean과 **동일한 밝은 배경** 유지 (다크 모드 금지)
- Google Fonts 페어링 허용 (display + body)
- 서브틀 그래디언트 (같은 팔레트의 밝은/어두운 셰이드)
- 다층 소프트 그림자, hover 리프트, entrance 애니메이션
- `::before`/`::after` 장식 요소 (액센트 보더, 뱃지, 오버레이)
- `prefers-reduced-motion` 존중 필수

### 3.4 서브탭 색상 상태

| 서브탭 | 미완료 | Ready | Hover | Active |
|--------|--------|-------|-------|--------|
| Wireframe | — | `#000` | `#333` | `#000` + 2px 하단선 |
| Clean | `#999` | `#64B5F6` | `#1E88E5` | `#1565C0` + 2px 하단선 |
| Polished | `#999` | `#81C784` | `#43A047` | `#2E7D32` + 2px 하단선 |

## 4. 타이포그래피 & 스페이싱

### 4.1 폰트 스택

| Phase | 폰트 | 용도 |
|-------|------|------|
| Phase 1 (Wireframe) | `-apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif` | 전체 |
| Phase 2 (Clean) | 동일 시스템 폰트 | 전체 |
| Phase 2 (Polished) | Google Fonts (`@import`) + 시스템 fallback | display: heading / body: 본문 |

### 4.2 타이포 스케일 (design-taste.md 기준)

| 용도 | 크기 | 굵기 | 행간 |
|------|------|------|------|
| 캡션 | 12px | 400 | 1.4 |
| 본문 | 14px | 400 | 1.5 |
| 큰 본문 | 16px | 400 | 1.5 |
| h3 | 20px | 600 | 1.3 |
| h2 | 24px | 600 | 1.2 |
| h1 | 32px | 600 | 1.2 |

### 4.3 스페이싱 시스템

**기본 단위**: 4px

| 토큰 | 값 | 용도 |
|------|-----|------|
| Warmth (소비자) | 24px container, 16px gap, 48px section | 넉넉한 여백, border-radius 8px |
| Precision (관리자) | 16px container, 8px gap, 24px section | 촘촘한 배치, border-radius 4px |

## 6. 컴포넌트 패턴

### 6.1 브라우저 프레임

```html
<div class="browser-frame wireframe" id="frame-optN">
  <div class="browser-chrome">
    <div class="browser-dots"><span></span><span></span><span></span></div>
    <div class="browser-url">yourapp.com/FEATURE_SLUG</div>
    <div class="browser-controls">- □ x</div>
  </div>
  <div class="browser-content">
    <!-- 와이어프레임 콘텐츠 -->
  </div>
</div>
```

| 속성 | 데스크톱 | 모바일 |
|------|---------|--------|
| 최대 너비 | `max-width: 900px` | `max-width: 390px` |
| 테두리 | `1px solid #ccc`, `border-radius: 8px` | `3px solid #333`, `border-radius: 24px` |
| 콘텐츠 오버플로 | `overflow: hidden` | `overflow: hidden` |
| 크롬 dots | 회색 (wireframe), 빨녹노 (Phase 2) | 숨김 |
| CSS 클래스 | `.browser-frame` | `.browser-frame.mobile-frame` |

### 6.2 탭 시스템

**메인 탭** (6개):
```
[ 1: Safe ] [ 2: Name ] [ 3: Name ] [ 4: Name ] [ 5: Name ] [ Summary ]
```
- `flex: 1` 균등 배분, `overflow-x: auto`
- 활성: `color: #000`, `border-bottom: 2px solid #000`, `font-weight: 600`

**서브 탭** (3개, 옵션당):
```
    ⬡           🖌          ◇
 Wireframe    Clean     Polished
```
- iOS 스타일 아이콘 탭바 (24x24 SVG + 12px 라벨)
- `justify-content: center`, `gap: 24px`
- wrapper 클래스를 `wireframe`/`clean`/`polished`로 토글

### 6.3 어노테이션 시스템

```html
<span class="wf-marker">①</span>  <!-- 인라인 마커 -->

<div class="wf-annotations">       <!-- 주석 목록 -->
  <h3>UX Notes</h3>
  <ul>
    <li>① 설명...</li>
  </ul>
</div>
```

- 마커: 18x18 원형, `border: 1px solid #ccc`, `font-size: 10px`
- Wireframe 서브탭에서만 표시 (Clean/Polished에서 `display: none`)

### 6.4 완료 배너

```html
<div class="completion-banner" id="completion-banner" style="display:none;">
  ✦ Visual designs complete — all options now have Clean + Polished variants
</div>
```

- JS `window.load` 시 `.sub-tab-badge` 표시 여부 체크
- 모든 배지 숨김 = 전체 완료 → 배너 8초 표시 후 자동 숨김
- `animation: bannerSlideDown 0.4s ease-out`

## 7. JavaScript 패턴

### 7.1 인라인 JS (template.html 내장)

모든 JS는 `<script>` 태그 하나에 인라인으로 포함. 외부 JS 파일 없음.

| 기능 | 구현 방식 |
|------|----------|
| 메인 탭 전환 | `querySelectorAll('.tab-btn')` → `classList.toggle` |
| 서브탭 전환 | `dataset.variant` → `frame.classList.remove/add` |
| 완료 배너 | `window.load` → badge visibility 체크 |

### 7.2 JS 제약사항

- ES5 호환 (화살표 함수, `const`/`let`, 템플릿 리터럴 사용 안함)
- `var`, `function`, `forEach` 패턴만 사용
- 외부 라이브러리 로드 없음
- Phase 1 생성 시 JS 수정 금지 (템플릿 그대로 유지)

## 8. 에이전트 파이프라인

### 8.1 2-Phase 워크플로우

```
Phase 1: UX Architect (Main Session, ~40-60s)
├── Step 1: Setup — brain/ 디렉토리 + 디자인 컨텍스트 확인
├── Step 2: First-Run — 코드베이스 탐색 + 스크린샷 수집 + design-context.md 생성
├── Step 3d: Generate — template 복사 → 편집 → 5개 B&W 와이어프레임
└── open index.html → 브라우저에서 즉시 확인

Phase 2: Visual Designer x5 (Parallel Task Agents)
├── Agent 1: styles-opt1.css (Option 1: Safe)
├── Agent 2: styles-opt2.css (Option 2)
├── Agent 3: styles-opt3.css (Option 3)
├── Agent 4: styles-opt4.css (Option 4)
└── Agent 5: styles-opt5.css (Option 5)
    각 에이전트: index.html 읽기 → #frame-optN 구조 분석
              → Clean (.clean .selector) 규칙 작성
              → Polished (.polished .selector) 규칙 작성
              → self-reveal CSS 추가
              → ≤ 200줄
```

### 8.2 에이전트 구성

| 에이전트 | subagent_type | 실행 방식 | 입력 | 출력 |
|---------|---------------|----------|------|------|
| UX Architect | main session | foreground | `$ARGUMENTS`, design-context.md, design-taste.md | index.html, base.css, styles.css, 5x 빈 styles-optN.css |
| Visual Designer x5 | `general-purpose` | parallel foreground | index.html, styles-optN.css, design-taste.md, design-context.md, self-reveal.css | styles-optN.css (≤ 200줄) |

### 8.3 옵션 생성 전략

| 옵션 | 역할 | UX 철학 |
|------|------|---------|
| **Option 1** | Safe | 기존 design-context.md 패턴 확장 (저위험 기준선) |
| **Option 2~5** | Exploratory | Progressive Disclosure, Dashboard-First, Wizard, Hub-and-Spoke, Card-Based, Conversational, Kanban, Timeline, Search-First 등에서 4개 선택 |

### 8.4 Summary 탭

- 5개 옵션 스코어링 테이블 (이름, 1줄 설명, 1~5점)
- 최적화 목표 기준 평가 (제공된 경우)
- 추천 옵션 + 1~2문장 근거
